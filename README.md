# Ink — Handwriting Journal (Native Android Build)

This repo holds the source code for the Ink journaling app, packaged as a native Android app via Capacitor. Every push to `main` triggers a build via GitHub Actions, producing a downloadable `.apk` file. No computer setup required.

---

## First-time setup (do this once)

### 1. Create the repo

You're already here if you're reading this on GitHub. ✓

### 2. Trigger the first build

The build runs automatically when you push to `main`. To trigger it manually right now:

1. Click the **Actions** tab at the top of this repo
2. Click **Build Android APK** in the left sidebar
3. Click the **Run workflow** dropdown (top right) → **Run workflow** (green button)
4. Wait 3–5 minutes for the build to finish

### 3. Download the APK

1. In the **Actions** tab, click the run that just completed
2. Scroll down to **Artifacts**
3. Click **ink-app-debug-apk** to download a ZIP
4. Extract the ZIP — inside is `app-debug.apk`

### 4. Install on your tablet

1. Transfer `app-debug.apk` to your tablet (Google Drive, USB, email — any way)
2. Open it on the tablet
3. Android will warn that it's from an unknown source. Allow it just for this app.
4. Tap **Install**. Tap **Open**.
5. Ink is now installed and shows up in your launcher.

### 5. Migrate your data from the old (PWA) version

If you've been using the Cloudflare-hosted version:

1. In the old PWA, tap **Export all as backup** in the sidebar footer
2. The JSON file lands in your tablet's Downloads folder
3. Open the new native Ink app
4. Tap **Import from backup** in the sidebar footer
5. Pick the JSON file
6. All notebooks and pages restore

## Day-to-day: making changes

You don't need any software on your computer for this. Everything happens in the GitHub web UI.

### To update the app

1. Edit `www/index.html` in GitHub's web editor (click the pencil icon)
2. Commit the change
3. GitHub Actions automatically builds a new APK (~3-5 min)
4. Download from **Actions → latest run → Artifacts**
5. Open the new APK on the tablet → Android replaces the old version in place
6. **Your data stays.** Android preserves the app's private storage across updates.

## Project layout

```
ink-app/
├── .github/workflows/build.yml    GitHub Actions build robot
├── www/
│   └── index.html                 The entire app — HTML + CSS + JS in one file
├── package.json                   Capacitor dependencies
├── capacitor.config.json          App identity (name, bundle ID, colors)
├── README.md                      You are here
└── android/                       Generated on first build, committed back later
```

The `android/` folder doesn't exist until the first build runs. After it's generated by `npx cap add android`, the GitHub Action commits it back so subsequent builds are faster.

## Troubleshooting

**Build fails on first run.** Look at the Action log. The most common first-run failure is the `npx cap add android` step needing extra permissions — re-run the workflow, it usually succeeds on retry.

**APK installs but won't open.** Long-press the icon → App info → Storage → Clear data, then reopen.

**Updated APK won't replace old one.** Both must be signed with the same key. For debug-signed APKs from this CI, this is automatic. If you ever switch to release-signed builds, you'll need to keep the keystore consistent.

**Lost data after reinstalling.** "Uninstall" (long-press → uninstall) deletes the app's private storage. "Replace via newer APK" doesn't. So always update via APK; only uninstall when you mean it.

---

## What this app does NOT do

- Auto-update from a server. You manually install new APKs.
- Sync to a cloud. All data is local to the tablet. Backups are your responsibility — export the JSON regularly and move it to Google Drive.
- Run on iOS. The architecture supports it via Capacitor, but no iOS build is set up.
